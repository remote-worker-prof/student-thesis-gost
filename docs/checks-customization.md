# Кастомизация проверок в форках

Документ для студентов и мейнтейнеров форков: где лежат правила и как менять их
без случайной поломки шаблона.

## Принцип безопасной кастомизации
1. Меняйте правила только в своем форке.
2. Меняйте один блок правил за раз и сразу прогоняйте соответствующий `make check-*`.
3. После локальной проверки прогоняйте полный `make check`.
4. Если ослабляете правило, зафиксируйте это в README своего форка.

<a id="style-rules"></a>
## STYLE rules (`scripts/check_tex_style.sh`)

Что можно менять:
- список проверяемых файлов (`collect_files`),
- запрещенные паттерны display-math,
- список блочных окружений, для которых требуются пустые строки.

Как ослабить:
- убрать конкретный паттерн из проверок `STYLE-001..003`,
- убрать окружение из regex в AWK-блоке (`STYLE-004`).

Как усилить:
- добавить новые паттерны,
- добавить новые окружения в проверку пустых строк.

<a id="intro-rules"></a>
## INTRO rules (`scripts/check_intro_structure.sh`)

Что можно менять:
- набор запрещенных inline-команд (`INTRO-001`),
- эталонные фразы обязательных блоков (`INTRO-002`),
- правила порядка блоков (`INTRO-003`),
- число задач и правила пунктуации (`INTRO-006`, `INTRO-007`).

Как ослабить:
- убрать отдельные запреты/блоки,
- разрешить другую длину списка задач,
- отключить проверку пунктуации.

Как усилить:
- добавить дополнительные обязательные фразы,
- добавить новые структурные условия (например, минимальный размер блока).

<a id="font-rules"></a>
## FONT rules (`scripts/check_fonts.sh`)

Что можно менять:
- regex допустимых имен шрифтов,
- список запрещенных fallback-шрифтов,
- набор обязательных семейств.

Как ослабить:
- расширить regex допустимых шрифтов (например, разрешить альтернативу в форке).

Как усилить:
- добавить проверку дополнительных семейств,
- добавить запрет на конкретные fallback-шрифты.

<a id="structure-rules"></a>
## STRUCT rules (`scripts/check_structure.sh`)

Что можно менять:
- список обязательных разделов (`required`).

Как ослабить:
- удалить или переименовать токены обязательных разделов.

Как усилить:
- добавить новые обязательные разделы/подразделы,
- проверять дополнительные фразы в приложениях.

<a id="layout-rules"></a>
## LAYOUT rules (`scripts/check_title_layout.sh`)

Что можно менять:
- порог `threshold` для RMSE,
- плотность растеризации,
- строгость зависимостей через `STRICT_LAYOUT_CHECK`.

Как ослабить:
- поднять порог RMSE,
- оставить `STRICT_LAYOUT_CHECK=0` по умолчанию.

Как усилить:
- снизить порог RMSE,
- всегда запускать в строгом режиме (`STRICT_LAYOUT_CHECK=1`).

## Карта «код правила -> место правки»

| Код(ы) | Где реализовано |
|---|---|
| `STYLE-*` | `scripts/check_tex_style.sh` |
| `INTRO-*` | `scripts/check_intro_structure.sh` |
| `FONT-*` | `scripts/check_fonts.sh` |
| `STRUCT-*` | `scripts/check_structure.sh` |
| `LAYOUT-*` | `scripts/check_title_layout.sh` |

## Общий формат сообщений
Единый формат выводится через:
- `scripts/check_common.sh`

Если хотите поменять шаблон текста ошибок сразу для всех проверок, начинайте с
этого файла.
